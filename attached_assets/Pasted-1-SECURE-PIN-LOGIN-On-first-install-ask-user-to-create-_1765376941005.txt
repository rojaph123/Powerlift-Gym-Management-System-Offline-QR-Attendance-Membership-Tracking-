3. REGISTRATION MODULE  
   Registering a person should include:
   - First name  
   - Last name  
   - Gender  
   - Age  
   - Email  
   - Phone  
   - Take profile photo  
   - Select **Membership Type**:
       student / regular / senior  
   - Select **Registration Option**:
       A. Member Only  
       B. Member + Monthly Subscription  
       C. Member + Per Session  

   **Rules:**
   - Membership (full membership) is yearly (lifetime QR).
   - Monthly subscription is separate.
   - Student • Regular • Senior → different monthly rates.
   - Session pricing depends on:
       - session_member  
       - session_nonmember  

   **Subscription Rules:**
   - If subscribing today:
       subscription_start = today  
       subscription_end = same day next month  
   - If expired:
       Example: expired March 1, today is March 5 → new expiry = April 5

   **Sales should record automatically** based on the choice.

4. QR SCAN MODULE (MUST BE FULLY IMPLEMENTED)
   Using mobile_scanner:
   - On successful scan:
       - Play beep sound
       - Validate QR format → must start with **GYM-**
       - Lookup member by qr_code
   - If no match:
       - Vibrate
       - Show red popup: “QR not recognized”
   - If found:
       Display:
         - Full name
         - Membership type (student / regular / senior)
         - Subscription status (active / expired)
         - Expiry date
         - Profile photo
   - Decision Flow:
     A. If subscription is ACTIVE → record attendance.
     B. If EXPIRED → ask:
        1. Renew monthly subscription  
        2. Pay per session  
     C. If Member-Only with no subscription → offer:
        - Monthly subscription  
        - Per session  
     D. If Non-member (walk-in, no QR):
        - Not stored in database  
        - Manual session sale only  

   **Required actions in QR scanner:**
   - Beep sound → assets/sounds/beep.mp3
   - Error vibration → HapticFeedback.mediumImpact()
   - Save record:
       - attendance table
       - sales table (if applicable)

   **Validations:**
   - Prevent double scan within 5 seconds.
   - Prevent scanning invalid QR images.
   - Detect QR image from gallery (optional fallback).
